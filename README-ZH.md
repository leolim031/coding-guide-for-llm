# LLM 编码指南

一套用于 **LLM 辅助开发** 的编码规范与行为准则。

本指南重点强调 **正确性、清晰性与可验证性**，同时避免 LLM 常见失误（过度重构、脑补需求、为了“短”牺牲可读性等），适合用于实际工程项目中的 AI coding workflow。

---

## 核心原则

- **正确性 > 清晰性 > 最小改动 > 性能 > 简洁性**
- 不要把“代码更短”当作目标
- 可读性与可测试性优先于技巧性写法
- 默认采用最小改动（surgical changes），但必要时必须修复根因
- 禁止添加未被明确要求的扩展功能或“未来优化”
- 任何非简单改动必须提供验证方式（测试或复现步骤）

---

## 适用场景

本指南适用于：

- 作为 LLM 的 system prompt / coding policy
- AI 辅助提交 PR 时的改动边界约束
- 团队协作中降低 AI 引入的 diff 风险
- 强制形成“修改后可验证”的工程闭环

---

## 使用方式

- 将 `instruction.md` 放入项目根目录或规范目录中
- 在 LLM prompt 或开发规范中引用该文件
- 作为代码审查参考标准

---

## 说明

本指南不是限制创造力，而是用于：

- 减少 LLM 过度发挥
- 降低无意义重构
- 提升修改可控性与可预测性
